# Тестування UI-компонентів веб-ресурсу Prostor.ua

Цей репозиторій містить результати лабораторної роботи з автоматизації тестування. Основна мета — побудова стабільної архітектури тестів за допомогою **Page Object Model (POM)**.
Об'єктом тестування обрано головну сторінку магазину **Prostor**. Написані тести покривають критичні точки взаємодії з користувачем: навігацію, пошукові елементи та зв'язки з соцмережами. 

Головна особливість реалізації — використання комбінованих селекторів (XPath та CSS) для забезпечення максимальної стабільності тестів до змін у верстці.
Цей проєкт містить набір із 10 автоматизованих тестів для перевірки критично важливих елементів головної сторінки інтернет-магазину **Prostor**.

## Технологічний стек
* **Мова програмування:** Python 3.x
* **Фреймворк:** Pytest
* **Інструмент автоматизації:** Selenium WebDriver
* **Паттерн проектування:** Page Object Model (POM)

## Структура:
* `locators.py` — містить класи з локаторами елементів (XPath, CSS Selectors).
* `prostor_page.py` — опис методів взаємодії зі сторінкою (Page Object).
* `test_prostor.py` — тестові сценарії (10).

## Test Cases:
1. **TC_001:** Перевірка заголовка сторінки (Title).
2. **TC_002:** Наявність головного логотипу бренду.
3. **TC_007:** Перевірка тексту та наявності кнопки "Каталог".
4. **TC_010:** Наявність номера гарячої лінії (0-800).
5. **TC_011:** Перевірка кнопки пошуку.
6. **TC_012:** Наявність посилання на Instagram.
7. **TC_013:** Наявність розділу "Акції".
8. **TC_014:** Перевірка посилання на Facebook у футері.
9. **TC_015:** Перевірка актуального копірайту (рік) у футері.
10. **TC_016:** Перевірка посилання на розділ "Доставка та оплата".
